Синхронизация

Таймаут соединения - Время ожидания отклика от сервера, миллисекунд.

Количество попыток соединения - Количество попыток, которые будут предприниматься для связи с сервером при проблемах с соединением.

Достраивать ветки - Достраивать ветки до корня.

Режим восстановления веток - Режим восстановления оборванных веток.

Локальная база пользователей - Создавать локальную копию базы пользователей (увеличивает трафик). Хранение основной информации о пользователях.

Пользователей в сессии – Максимальное количество пользователей, забираемое за один раз. Всего придётся скачать несколько десятков тысяч пользователей. Поэтому если за раз трафик не жалко выставляйте большое число. Оптимальным на мой взгляд является значение 50.

Сообщений в сессии - Максимальное количество сообщений, забираемое за один раз. -1 без ограничения.

Стоит иметь в ввиду, что на сервере ограничен размер передаваемых данных за одну синхронизацию и при длительном перерыве для выкачивания всех сообщений необходимо сделать несколько синхронизаций подряд.

Пометка своих сообщений - Помечать свои сообщения как прочитанные. Все собственные сообщения автоматически метятся как прочитанные.

Автосинхронизация - Автоматически запускать синхронизацию через указанный промежуток времени.

Интервал автосинхронизации - Интервал, по прошествии которого будут производиться попытки автосинхронизации, секунд.

Показывать индикатор прогресса - Показывать индикатор прогресса на форме синхронизации. Полезно на медленном интернете – видно сколько ещё ждать осталось.

Использовать сжатие - Использовать сжатие при получении данных с сервера. Значительно экономится трафик.

Сжатие gzip должны поддерживать ваши прокси-сервер и файервол (если вы их используете). Подробности о проблеме с Outpost Firewall

Прокси-сервер

Настройки прокси-сервера.

Эти настройки относятся только к синхронизации. За показ внешних картинок внутри приложения отвечают настройки прокси-сервера в Internet Explorer.
Внимание. Они обладают присущей .NET приложениям особенностью: если настройки прокси не указанны, то берутся настройки по умолчанию из свойств Internet Explorer.

Использовать прокси-сервер - Использование прокси-сервера при соединении с Webservice'ом.

Использовать авторизацию - Использование авторизации для соединений через прокси-сервера. Запрос авторизации появляется при соединении.

Настройки прокси-сервера

Адрес - Адрес прокси-сервера, например "127.0.0.1".

Порт - Порт прокси-сервера, например "3128".
